1ภาพรวมระบบ

Dire Wolf OS คือระบบจัดการโปรเจกต์และงานภายในองค์กร พัฒนาขึ้นสำหรับทีม Wolvester Marketing โดยเฉพาะ รองรับการทำงานแบบ Multi-role ตั้งแต่ Admin ไปจนถึง Viewer สามารถเข้าใช้งานได้ผ่าน Browser ทุกอุปกรณ์ ทั้ง Desktop, Tablet และ Mobile โดยไม่ต้องติดตั้งแอปพลิเคชันเพิ่มเติม

ระบบทำงานแบบ Single Page Application (SPA) เชื่อมต่อกับ Supabase Database แบบ Real-time ข้อมูลทุกอย่างถูกบันทึกลง Cloud โดยอัตโนมัติ

จุดเด่นสำคัญ

✦ จัดการโปรเจกต์และงานในที่เดียว  ·  ✦ Sprint Timeline แบบ Gantt  ·  ✦ ระบบสิทธิ์ตาม Role  ·  ✦ ใช้งานได้ทุกอุปกรณ์  ·  ✦ รองรับภาษาไทย

4เข้าสู่ระบบ

  1. เปิด Browser และไปที่ https://labs-d.online/
  2. กรอก Username หรือ Email ของคุณ
  3. กรอก Password (ค่าเริ่มต้นที่ Admin กำหนดให้)
  4. กด Sign In เพื่อเข้าสู่ระบบ
🔑

หลังเข้าระบบครั้งแรก ควรเปลี่ยนรหัสผ่านทันที ไปที่เมนู Team → กด Change password ในส่วน My Account

เปลี่ยนรหัสผ่าน

  1. คลิกเมนู Team ใน Sidebar
  2. ส่วน My Account ด้านบน → กด Change password
  3. กรอกรหัสผ่านปัจจุบัน → รหัสใหม่ → ยืนยันรหัสใหม่
  4. กด Save password
⚠️

Session จะหมดอายุเมื่อปิด Tab หรือ Browser หากต้องการใช้งานต่อต้อง Login ใหม่

5หน้า Overview (Dashboard)

หน้าหลักแสดงภาพรวมสถานะงานของทั้งบริษัท ประกอบด้วย:

สิ่งที่แสดงในหน้า Overview

Metric Cards — สรุปจำนวนงานแยกตาม Status (Backlog / In Progress / Review / Approved)

Sprint Timeline — ไทม์ไลน์แสดงงานทุกชิ้นในรูปแบบ Gantt Chart คลิกงานเพื่อดูรายละเอียด

Priority Tasks — งานที่มี Priority สูงและ Due date ใกล้ถึง

Department Filter — กรองมุมมองตาม Department โดยคลิก Department Pill ใน Sidebar

Sprint Timeline

Timeline แสดงงานทั้งหมดเรียงตามวันเริ่มต้น-สิ้นสุด สามารถสลับมุมมองได้:

Day Week Month Quarter

คลิกที่แถบงานใด ๆ ใน Timeline เพื่อเปิด Drawer รายละเอียดงานนั้นทันที

6จัดการโปรเจกต์

สร้างโปรเจกต์ใหม่

  1. คลิก + New Project (ปุ่มมุมขวาบน) หรือ + New Task แล้วเลือก New Project
  2. กรอกชื่อโปรเจกต์ และเลือก Department
  3. ตั้ง Start Date และ End Date เพื่อให้ Sprint Timeline แสดงผลถูกต้อง
  4. เลือก Status (Active / On Hold / Completed)
  5. หากต้องการซ่อนจาก Role ต่ำกว่า Executive ให้เปิด Secret Project
  6. กด Save

Sprint Timeline ของโปรเจกต์

เมื่อเปิด Drawer โปรเจกต์ จะเห็น Sprint Timeline แสดงงานทั้งหมดในโปรเจกต์นั้น โดยใช้ Start Date ของโปรเจกต์ เป็นจุดเริ่มต้นของ Timeline (ไม่ใช่วันนี้)

💡

การตั้ง Start Date ให้โปรเจกต์มีความสำคัญมาก หากไม่ตั้ง Timeline จะ Default เป็นวันนี้ ทำให้ดูงานในอดีตไม่ได้

ลบโปรเจกต์

🗑️

การลบโปรเจกต์จะ ลบงาน (Task) ทุกชิ้นในโปรเจกต์นั้นออกจาก Database ด้วย ไม่สามารถกู้คืนได้ ระบบจะแจ้งจำนวนงานที่จะถูกลบก่อนยืนยัน

7จัดการงาน (Task)

สร้างงานใหม่

  1. คลิก + New Task มุมขวาบน
  2. กรอก ชื่องาน (จำเป็น)
  3. เลือก Project ที่งานนี้สังกัด
  4. กำหนด Assignee, Due Date, Priority
  5. ตั้ง Status และ Progress (0–100%)
  6. กด Save Task

Subtask

งานย่อยภายในงาน แต่ละ Subtask มี Title, Status และ Progress เป็นของตัวเอง เมื่อมี Subtask ระบบจะ คำนวณ Progress ของงานหลักจาก Subtask โดยอัตโนมัติ (ค่าเฉลี่ย) โดยไม่ใช้ Slider ของงานหลัก

Comments

คลิก Show ที่ Panel ความคิดเห็นใน Drawer งาน เพื่อขยายดูและเพิ่ม Comment ได้ Panel จะปิดโดยอัตโนมัติหากยังไม่มี Comment ใด ๆ

10Calendar

มุมมองปฏิทินแสดง Due date ของงานทุกชิ้น คลิกวันที่ใดวันหนึ่งเพื่อดูรายการงานที่ครบกำหนดในวันนั้น คลิกงานเพื่อเปิดรายละเอียด

💡

งานที่ไม่มี Due date จะไม่ปรากฏใน Calendar แนะนำให้ตั้ง Due date ทุกครั้งที่สร้างงาน

11Meetings

บันทึกการประชุมใหม่

  1. ไปที่เมนู Meetings → คลิก + New Meeting
  2. กรอกหัวข้อ (Title), วันที่, เวลา และสถานที่
  3. เพิ่ม Agenda รายการวาระการประชุม
  4. เลือก Attendees จากรายชื่อในระบบ
  5. หลังประชุม เพิ่ม Minutes (สรุปผลการประชุม) และ Action Items
  6. แนบไฟล์เอกสารประกอบได้ใน Attachments
  7. กด Save

13Team & ผู้ใช้

  1. ไปที่เมนู Team
  2. คลิก + Add User เพื่อเพิ่มผู้ใช้ใหม่

  4. เลือก Department ที่สังกัด

System Log (Admin & CEO)

ด้านล่างสุดของหน้า Team แสดง Log การ Login ย้อนหลัง 7 วัน บันทึก Username, เวลา และ IP address โดย Log เก่ากว่า 7 วันจะถูกลบอัตโนมัติ

14การแจ้งเตือน (Notifications)

ระบบสร้าง Notification อัตโนมัติเมื่อ:

  • มีงานใหม่ที่ถูก Assign ให้คุณ
  • งานที่คุณเกี่ยวข้องมีการเปลี่ยน Status
  • งานใกล้ถึง Due date
  • มีการ Comment ในงานที่คุณรับผิดชอบ

Badge ตัวเลขสีแดงบนเมนู Notifications แสดงจำนวนการแจ้งเตือนที่ยังไม่ได้อ่าน

16ไฟล์แนบ (Attachments)

สามารถแนบไฟล์ได้ใน Task, Project, Meeting และ CEO Brief ไฟล์จะถูกบันทึกลง Database โดยตรง

17ความปลอดภัย

มาตรการรักษาความปลอดภัยที่ใช้

✅ Password Hashing — รหัสผ่านถูกเข้ารหัส bcrypt ก่อนบันทึกลง Database

✅ XSS Protection — ข้อมูลที่ User กรอกทั้งหมดผ่าน escapeHtml() ก่อนแสดงผล

✅ Session Isolation — Session หมดอายุเมื่อปิด Browser ไม่มีข้อมูลค้างใน Local Storage

✅ Role-based Access — ทุก Action ตรวจสิทธิ์ Role ก่อนดำเนินการ

🔐

คำแนะนำสำหรับ Admin: ตั้งรหัสผ่านเริ่มต้นที่ซับซ้อน และแจ้งให้ User ทุกคนเปลี่ยนรหัสผ่านหลัง Login ครั้งแรก อย่าใช้รหัสผ่านเดียวกันหลาย Account
